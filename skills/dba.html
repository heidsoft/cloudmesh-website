场景一：慢查询优化

性能优化
你:@ClawOps 生产库 order 表查询变慢了，帮我分析下
ClawOps:正在分析 order 表慢查询...
📊 慢查询 TOP 3： 1️⃣ SELECT * FROM order WHERE create_time > ? 执行时间：3.2s | 扫描行数：850万 问题：无索引，全表扫描 建议：添加索引 idx_create_time 2️⃣ SELECT * FROM order WHERE user_id = ? AND status = ? 执行时间：2.8s | 扫描行数：420万 问题：单字段索引效率低 建议：联合索引 idx_user_status 💡 预计优化后查询时间 < 100ms 需要我生成 ALTER 语句吗？

场景二：容量预警

容量管理
你:@ClawOps 检查生产数据库容量
ClawOps:正在分析数据库容量...
📊 生产数据库容量报告 存储空间： • 已用：1.2TB / 2TB (60%) • 日均增长：15GB • 预计满载：53 天后 ⚠️ 预警： • order_log 表占比 45%（可归档） • binlog 保留 7 天（可减少至 3 天） 💡 建议： 1. 归档 30 天前日志，释放约 400GB 2. 调整 binlog 保留策略 3. 计划 Q2 扩容 需要我执行归档操作吗？

场景三：DDL 变更审核

变更管理
你:@ClawOps 审核这个 DDL：ALTER TABLE user ADD COLUMN phone VARCHAR(20)
ClawOps:正在审核 DDL 语句...
📋 DDL 变更审核报告 表：user（用户表） 当前行数：1200万 表大小：8.5GB ✅ 语法检查：通过 ✅ 字段命名：符合规范 ⚠️ 风险评估：中等风险 风险点： • 大表 DDL 可能锁表 5-10 分钟 • 业务高峰期影响用户 建议： 1. 使用 pt-online-schema-change 在线变更 2. 在业务低峰期（02:00-04:00）执行 3. 提前通知业务方 已生成在线变更脚本，是否需要执行？
