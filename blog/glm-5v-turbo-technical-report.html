GLM-5V-Turbo 技术报告：多模态 Agent 基座模型深度解读

从 CogViT 到 MMTP，智谱如何打造多模态 Agent 认知内核

2026-05-12 智谱开放平台 多模态 · Agent · 基座模型

背景

现实世界中的任务天然是多模态的——人类每天同时处理文字、图表、截图、网页等信息。如今大模型正从纯语言理解转向多模态 Agentic 能力，不仅要求模型理解图像、视频、网页、GUI，还需要在此基础上完成规划与行动。

GLM-5V-Turbo 是智谱发布的新一代多模态基座模型，在多模态 Coding、Tool Use、GUI Agent 等方面取得了极具竞争力的性能。

核心技术改进

1. CogViT：参数高效的视觉编码器

从零构建的视觉编码器，采用两阶段预训练：

第一阶段：掩码图像建模（MIM）

  • 输入 224×224，35% 掩码率重建被遮挡区域
  • 同时对齐两个教师模型特征：SigLIP2（语义表示）+ DINOv3（纹理特征）
  • 数据混合：80% 高质量自然图像 + 10% 指令跟随 + 10% 科学图像
  • 优化器使用 Muon，引入 QK-Norm 提升训练稳定性

第二阶段：对比式图文预训练

  • 用 NaFlex 方案替代固定 224×224 输入，支持变尺寸、保留原始长宽比
  • SigLIP loss 全局批次扩到 64K
  • 80 亿规模中英文双语图文语料
📊

性能表现（403M参数的 CogViT-L）

基准 分数
ImageNet-1K 零样本 83.5

多数指标超过 SigLIP2-SO (427M) 和 DFN-H (632M)，以更小参数实现更优性能。

2. MMTP：多模态多 Token 预测

传统 MTP 中，前缀 token 可直接输入 MTP head，但视觉 token 如何传递是个难题。团队对比了 3 种方案：

方案对比

方案 1：直接传视觉嵌入 → 通信复杂度高，需要在流水线并行的多个 stage 之间传递视觉嵌入

方案 2：完全掩码视觉 token → 退化为纯文本 MTP，丢失多模态特性

方案 3：视觉位置信息 + 共享可学习图像占位 token → ✅ 最终采用

💡

第 3 种方案优势

  • 不需要在流水线并行的 stage 间传递视觉嵌入，显著降低通信复杂度
  • 训练损失更低、收敛更稳定
  • 天然兼容序列并行和上下文并行，无需额外处理视觉嵌入的分区与对齐

3. 广覆盖联合训练

预训练阶段深度融合视觉与语言，覆盖：

世界知识、图文交错、OCR、编程、GUI、视频、多模态工具使用、空间感知、grounding、学科问题求解……特别加大了多模态编程数据的比例，并在 30+ 任务类别下进行联合 RL 优化。

几个关键发现

  • 多任务 RL 跨域干扰比 SFT 更弱，多领域均获得稳定增益
  • 跨域协同训练让分布窄、单任务易震荡的领域优化更稳定
  • 思维模式存在跨任务迁移：一个领域学到的推理行为有时能在另一个领域带来可衡量的收益

4. 大规模多模态 RL 基础设施

从 4 个维度重构训练栈：

  • 统一任务与奖励抽象：VLM RL Gym 提供统一接口，奖励系统集中编排多验证器
  • 全管线解耦与异步：rollout 推理、奖励评估、batch 构建、权重传输四阶段最大化重叠
  • 面向多模态的细粒度内存管理：为 ViT 与 projector 设计独立内存策略，结合细粒度定向重计算与 CPU offload
  • 拓扑感知的视觉输入分区：减少约 7GB GPU 通信缓存开销

框架集成：感知-规划-执行闭环

GLM-5V-Turbo 可作为认知核心嵌入 Claude Code、AutoClaw 等外部 Agent 框架：

三角协作架构

  • Claude Code：负责终端环境与本地文件系统的执行逻辑
  • AutoClaw：提供浏览器与 GUI 自动化的"双手"
  • GLM-5V-Turbo：承担视觉-语言控制器的角色

三者构成完整的「感知-规划-执行」闭环。

ImageMining 基准

专门测试"用图像思考、用图像深度搜索"能力的视觉中心基准，要求模型通过多步工具调用主动从图像中"挖"出线索。

  • 217 道人工整理测试题，覆盖社交、娱乐、商品、地点、富文本、自然、科学领域
  • 对应通用识别、时空推理、事件推理、富文本推理、视觉搜索推理任务
  • 引入 Visual Jump 约束（WEB_VISUAL），强制中间推理步骤必须涉及视觉跳转
  • 已开源，欢迎行业交流

总结

GLM-5V-Turbo 的核心突破在于：

  1. CogViT 以更小参数（403M）实现更优性能，两阶段预训练兼顾语义与纹理
  2. MMTP 解决视觉 token 传给 MTP head 的难题，图像占位 token 方案兼顾效率与效果
  3. 广覆盖联合训练 让多任务 RL 跨域增益成为可能，思维模式可跨任务迁移
  4. 大规模 RL 基础设施 为多模态训练提供工程保障，4 个维度重构训练栈
  5. 生态集成（Claude Code + AutoClaw）证明模型可作为多模态 Agent 的认知内核

GLM-5V-Turbo 进行了 4 个方面的技术改进：新视觉编码器 CogViT、对大规模基础设施友好的多模态多 token 预测 MMTP、覆盖感知-推理-Agent 的广覆盖联合训练，以及一套面向大规模多模态强化学习的基础设施。
