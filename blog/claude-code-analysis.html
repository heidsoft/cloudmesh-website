一、项目概述

Claude Code 是 Anthropic 推出的终端 AI 编程助手，通过 npm source map 泄露事件于 2026年3月31日被公开。这是目前已知的最完整的 AI Agent CLI 源码，包含约 1,900 个文件、512,000+ 行 TypeScript 代码。

📌 来源：heidsoft/claude-code 项目为研究目的整理的源码快照
🛠️ 技术栈：Bun 运行时 + React + Ink (终端 UI 框架) + TypeScript

三、核心模块详解

1. 入口模块 (main.tsx)

CLI 入口、启动初始化、命令路由。启动时做了大量性能优化：

启动性能优化 - 并行执行
// 这些副作用必须在其他导入之前运行：
// 1. profileCheckpoint 标记入口点
// 2. startMdmRawRead 启动 MDM 子进程并行执行
// 3. startKeychainPrefetch 并行读取 keychain (OAuth + API Key)
profileCheckpoint('main_tsx_entry');
startMdmRawRead();
startKeychainPrefetch();

优化效果：MDM + Keychain 并行执行，节省约 200ms 启动时间。

2. 命令系统 (commands.ts)

注册和管理所有 slash 命令，共支持约 50+ 个命令。

分类命令说明
Git/commit, /diff, /branchGit 操作
调试/doctor, /review, /test诊断和审查
上下文/context, /compact, /memory上下文管理
会话/resume, /share, /teleport会话管理
任务/tasks, /plan任务管理
配置/config, /theme, /mcp配置管理

3. 工具系统 (tools.ts)

定义 Agent 可调用的工具能力，共约 40+ 个工具。

类别工具说明
文件操作BashTool, FileReadTool, FileWriteTool, FileEditTool, GlobTool, GrepToolShell执行、文件读写、搜索
网络WebFetchTool, WebSearchToolURL获取、Web搜索
任务与AgentAgentTool, TaskCreateTool, TaskUpdateTool子Agent spawn、任务管理
高级LSPTool, McpTool语言服务器、MCP协议

4. 查询引擎 (QueryEngine.ts)

核心交互循环，处理用户输入、调用 LLM、执行工具。

用户输入 → 构建 Context → Anthropic API → 解析 Tool Use → 执行工具 → 循环

上下文构建包括：System Prompt、用户上下文（当前目录、项目结构）、对话历史、工具执行结果。

5. 服务层 (services/)

外部集成和能力抽象，包括：

  • API 服务：Anthropic API 客户端、文件上传下载
  • MCP 服务：Model Context Protocol 客户端
  • 分析服务：GrowthBook 特性开关、使用量统计
  • 插件系统：插件加载器、内置插件初始化
  • 上下文压缩：对话上下文压缩、Token 成本追踪

四、关键技术特性

1. Feature Flag 机制

通过 bun:bundle 的 feature 函数实现条件编译，tree-shaking 移除未启用的代码：

条件导入示例
import { feature } from 'bun:bundle';

const voiceCommand = feature('VOICE_MODE')
  ? require('./commands/voice/index.js').default
  : null

支持的特性：VOICE_MODE、KAIROS、BUDDY、BRIDGE_MODE、COORDINATOR_MODE

2. Lazy Import 打破循环依赖

延迟加载
const getTeammateUtils = () => require('./utils/teammate.js');

3. MCP (Model Context Protocol)

支持通过 MCP 协议动态扩展 Agent 能力，实现工具和资源的动态发现。
