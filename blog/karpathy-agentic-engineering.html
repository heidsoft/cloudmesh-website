Karpathy 谈 AI 编程未来：Agentic Engineering 是核心竞争力

深度解读 2026年4月访谈：Vibe Coding 抬高下限，Agentic Engineering 守住上限

2026年5月7日 10 分钟阅读

2026年4月，Andrej Karpathy（安德烈·卡帕蒂）接受了深度访谈。他是前 OpenAI 联合创始人、Tesla Autopilot 视觉团队负责人，在 AI 圈子里有极高的分量。

这场访谈的核心主题只有一句话：AI 编程的下一步不再是"更快写代码"，而是通过"智能体工程"守住软件的质量、安全和责任门槛。

两个核心概念

Vibe Coding（凭感觉编程）

利用自然语言让 AI 生成代码，开发者不再逐行编写或修改细节。

本质：抬高了下限——让非专业者也能创造软件。但代价是代码可能"能跑但很难看"。

Agentic Engineering（智能体工程）

设计、协调和监督 AI Agent 的工程学科，确保系统质量。

本质：守住上限——在利用 AI 速度的同时，必须守住专业软件的安全、架构和审美标准。

⚖️

关键平衡

Vibe Coding 让每个人都能做软件（民主化），但 Agentic Engineering 要求开发者必须理解系统结构，防止 AI 写出有安全隐患的代码（专业化）。

例子：AI 可能会用"邮箱地址"来绑定支付资金，这在工程上是危险的——用户可能用不同邮箱登录和支付。必须由人类设定规则使用"内部唯一 ID"。

Software 3.0：上下文即程序

Karpathy 提到了一个深刻的概念转变：

旧范式 新范式（Software 3.0）
写 Shell 脚本适配各种环境 把安装说明复制给 Agent，让它根据你的机器环境自行执行和调试
代码膨胀难以维护 你不再只是操作代码编辑器，而是在操作 LLM 的 Context Window（上下文窗口）

这意味着程序的边界扩大了——现在包括 Prompt、文件、工具调用和日志。

编程不再是写显式代码，而是构造 Context（上下文）。你的工作从"写代码"变成了"设计环境让 AI 帮你写代码"。

锯齿状智能：LLM 的真实局限

Karpathy 强调了一个反直觉的事实：LLM 是"幽灵"而非"动物"——它们由数据和奖励函数塑造，没有动机和情绪。

反直觉案例

最先进的模型可能能重构 10 万行代码或找到零日漏洞，却可能在"去 50 米外洗车是走路还是开车"这种常识题上犯错。

原因：它忽略了"车需要被洗"这个前提。

关键教训：

  • 你不能对 LLM 大吼大叫让它变聪明
  • 必须通过工程手段（如 Agentic Engineering）来规避其弱点
  • 不能因为模型在某方面强就盲目信任，必须明确其能力边界

开发者的转变

Karpathy 提到，2025年12月是他个人的转折点：

之前 之后
需要修补 AI 生成的代码 AI 输出的代码直接可用，进入 Vibe Coding 状态
开发者核心能力：记忆 API 细节 开发者核心能力：指挥一组会犯错的 Agent

他说记不清上一次修改 AI 代码是什么时候了，但他必须负责顶层设计（Spec）。

对创业者的预测

创业机会在哪里？

寻找那些尚未被 RL（强化学习）覆盖的"可验证"领域。如果一个问题可以被验证（有明确的对错标准），就可以通过 RL 训练模型解决。

这意味着：垂直行业的业务逻辑验证是创业者的机会。

教育重点的转变

智能变便宜后，"理解"变得昂贵。

  • 你可以外包思考步骤
  • 但不能外包对系统的理解
  • 学习的重点应从记忆细节转向理解底层机制（Tensor 原理、系统架构）

AI-Native 工程师面试变了

未来的面试不该考算法题，而应该：

给一个大项目（如做一个安全的 Twitter 克隆），然后用 10 个 Cursor Agent 去攻击它，看候选人能否构建出经得起攻击的系统。

对我们的启示

Karpathy 的观点和我们在研究 Claude Code Skills 时发现的趋势高度吻合：

Karpathy 的概念 对应的 Skill/工具
Agentic Engineering obra/superpowers (180k ⭐) — Agentic Skills 框架
Vibe Coding 各种 AI Coding 工具，抬高下限
Software 3.0 = Context Prompt + 文件 + 工具调用 = 程序边界扩大
Jagged Intelligence humanizer — 解决 AI 输出"机器味"问题

核心结论：

做产品时，应该做 Agentic Engineering 层面的工具，而不是 Vibe Coding 层面的工具。

前者守住上限，后者抬高下限。前者有商业价值，后者只有普惠价值。

AI 的未来不是取代人类

人类需要进化出新的工程纪律来驾驭这些强大但不完美的"幽灵"。
